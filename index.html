🚀 Cloud Delivery Pipeline Portafolio

Proyecto orientado a demostrar un flujo de entrega con enfoque DevOps, integrando Integración Continua, validación de calidad, control de Pull Request y despliegue simulado como evidencia de un pipeline real.

Ver repositorio Ver arquitectura

🎯 Objetivo

Implementar un pipeline que permita detectar errores de forma temprana, asegurar calidad de código, validar cobertura de pruebas y mantener control sobre cambios hacia producción.

Qué demuestra

  • Automatización de validaciones técnicas
  • Control de calidad antes del merge
  • Estructura clara de CI y CD
  • Publicación visible para portafolio

Valor entregado

  • Menor riesgo de errores en integración
  • Mayor consistencia en el proceso
  • Trazabilidad del flujo de entrega
  • Evidencia concreta de prácticas DevOps

⚙️ Arquitectura del Pipeline

El flujo implementado separa correctamente la validación técnica, la calidad del código y la publicación del contenido del portafolio.

Pull Request / Push
CI: Tests + Coverage
Quality Gate con Flake8
Merge controlado a main
CD Simulado
GitHub Pages

🔍 Componentes implementados

Integración Continua

  • Instalación automatizada de dependencias
  • Ejecución de pruebas con pytest
  • Medición de cobertura con pytest-cov

Quality Gate

  • Validación de errores críticos
  • Control de calidad con flake8
  • Bloqueo lógico antes del avance del flujo

CD Simulado

  • Generación de artefacto de despliegue
  • Registro de ejecución y trazabilidad
  • Representación de release controlado

GitHub Pages

  • Publicación visible del proyecto
  • Separación entre pipeline y capa de presentación
  • Consumo simple para reclutadores y entrevistas

✅ Estado actual

El proyecto se encuentra publicado y visible, con validaciones activas de CI y calidad, manteniendo GitHub Pages como vitrina del portafolio sin interferencia del pipeline.
