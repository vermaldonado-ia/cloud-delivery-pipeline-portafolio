💡 Propósito del proyecto

Este portafolio fue desarrollado para evidenciar capacidades de estructuración, control y automatización de un flujo DevOps, con foco en calidad, trazabilidad y visibilidad del proceso de entrega.

✅ Integración Continua

El pipeline ejecuta validaciones automáticas sobre el código para detectar errores de forma temprana antes de integrar cambios.

✅ Control de calidad

Se incorporan pruebas automatizadas, métricas de cobertura y validaciones de calidad como mecanismo de control previo al despliegue.

✅ Evidencia visible

La publicación en GitHub Pages permite mostrar de manera clara el resultado del flujo de entrega y la trazabilidad del proyecto.

⚙️ Flujo del pipeline

El proyecto representa una secuencia lógica de validación y entrega, similar a un flujo real de CI/CD.

Paso 1

Pull Request / Push

Ingreso controlado de cambios al repositorio.

Paso 2

CI Pipeline

Instalación, validación de entorno y ejecución de pruebas automatizadas.

Paso 3

Quality Gate

Revisión de calidad y cobertura antes de continuar.

Paso 4

Merge controlado

Integración a main solo si las validaciones aprueban.

Paso 5

CD Simulado

Preparación de release y publicación del resultado como evidencia.

Resultado esperado: detectar fallas antes del despliegue, mejorar la confiabilidad del proceso y demostrar un enfoque moderno de entrega continua.

🛠️ Componentes implementados

El repositorio integra distintas prácticas técnicas y de gobernanza del flujo de entrega.

CI Pipeline

  • Checkout del repositorio
  • Configuración de Python
  • Instalación de dependencias
  • Ejecución de pruebas con pytest
  • Generación de métricas de coverage

Quality Gate

  • Validación de cobertura mínima
  • Bloqueo de cambios defectuosos
  • Control previo al merge
  • Base para integración con herramientas como SonarQube

Control de Pull Request

  • Validación automática antes de integrar cambios
  • Protección de rama principal
  • Mayor trazabilidad del flujo de trabajo

Publicación en GitHub Pages

  • Visualización pública del proyecto
  • Presentación ejecutiva del pipeline
  • Evidencia visible para portafolio profesional
