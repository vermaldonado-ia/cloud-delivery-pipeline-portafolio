💡 Propósito del proyecto

Este portafolio fue desarrollado para evidenciar capacidades de estructuración, automatización y control del flujo de entrega de software, con foco en calidad, trazabilidad y visibilidad del proceso.

✅ Integración Continua

El pipeline ejecuta validaciones automáticas sobre el código para detectar errores de forma temprana antes de integrar cambios.

✅ Control de calidad

Se incorporan pruebas automatizadas, métricas de cobertura y validaciones de calidad como mecanismo de control previo al despliegue.

✅ Evidencia pública

La publicación en GitHub Pages permite mostrar el resultado del proyecto de manera visible, profesional y fácil de compartir.

⚙️ Flujo del pipeline

El proyecto representa una secuencia lógica de validación y entrega, alineada con un flujo real de CI/CD.

Paso 1

Pull Request / Push

Ingreso controlado de cambios al repositorio.

Paso 2

CI Pipeline

Configuración de entorno, instalación y ejecución de pruebas.

Paso 3

Quality Gate

Control de calidad y cobertura antes de aprobar el cambio.

Paso 4

Merge a main

Integración controlada solo si las validaciones aprueban.

Paso 5

CD Simulado

Preparación del release y publicación visible como evidencia.

Este flujo permite detectar fallas antes del despliegue, fortalecer la confiabilidad del proceso y demostrar una base sólida de prácticas DevOps.

🛠️ Componentes implementados

El repositorio integra distintas prácticas técnicas y de control del flujo de entrega.

CI Pipeline

  • Checkout del repositorio
  • Configuración de Python
  • Instalación de dependencias
  • Ejecución de pruebas con pytest
  • Generación de métricas de coverage

Quality Gate

  • Validación de cobertura mínima
  • Bloqueo de cambios defectuosos
  • Control previo al merge
  • Base para evolución hacia SonarQube

Pull Request Control

  • Validación automática antes de integrar cambios
  • Mayor trazabilidad del flujo de trabajo
  • Protección de la rama principal

GitHub Pages

  • Publicación visible del proyecto
  • Presentación ejecutiva del pipeline
  • Evidencia pública para portafolio profesional
