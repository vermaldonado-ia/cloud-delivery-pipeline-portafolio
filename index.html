🚀 Cloud Delivery Pipeline Portafolio

Proyecto orientado a demostrar la implementación de un flujo de entrega moderno, integrando Integración Continua (CI), validación de calidad, control de cambios mediante Pull Request y un despliegue simulado (CD) con publicación en GitHub Pages.

Ver repositorio Ver pipeline

🎯 Objetivo

Mostrar de forma práctica cómo se puede estructurar un pipeline de validación y control de cambios, fortaleciendo la calidad del software y la trazabilidad del proceso.

⚙️ Flujo implementado

  • Pull Request / Push
  • CI Pipeline
  • Quality Gate
  • Merge controlado a main
  • CD (Deploy en GitHub Pages)
  • Publicación en GitHub Pages

🛠️ Componentes

  • GitHub Actions
  • Python
  • Pytest
  • Coverage
  • Control de Pull Request
  • GitHub Pages

📌 Valor del proyecto

Este portafolio evidencia capacidades de automatización, control de calidad, trazabilidad y visibilidad del proceso de entrega, aplicables a contextos reales de transformación tecnológica.

Desarrollado como evidencia de portafolio profesional en prácticas de entrega continua y control de calidad.
